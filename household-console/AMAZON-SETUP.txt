Amazon / AWS connections for Home console (household-console)
================================================================

This app is static HTML/CSS/JS. "Connections" mean: (1) host it on AWS Amplify,
(2) open it on Fire TV, and (3) optionally wire Supabase at build time.

Build (repo root amplify.yml)
-------------------------------
  • Commands call scripts/amplify-prebuild.sh and scripts/amplify-verify.sh so
    the buildspec avoids bare ":" characters in YAML (Amplify rejects those as
    malformed unless every such command is fully quoted).
  • preBuild: optionally writes household-console/shared/sync-config.json when
    SUPABASE_URL and SUPABASE_ANON_KEY are set in Amplify environment variables.
  • build: verifies tv/index.html, manage/index.html, and shared/store.js exist,
    then publishes the household-console/ folder as the site root (no compile step).

To sanity-check locally from the repository root (same as Amplify):

  bash scripts/amplify-prebuild.sh && bash scripts/amplify-verify.sh


1) AWS Amplify Hosting (recommended)
------------------------------------
In AWS Console (https://console.aws.amazon.com/amplify/):

  a) Create new app → "Host web app" (or "Deploy without Git" to upload a zip).

  b) Connect your Git provider and select this repository.

  c) Build settings: Amplify should detect amplify.yml at the repo root. The
     build publishes the household-console/ folder as the site root.

  d) Save and deploy. Note the HTTPS URL, for example:
       https://main.d1234567890abc.amplifyapp.com/

  e) Paths on that host:
       /              → landing (index.html)
       /tv/           → Fire TV / 10-foot dashboard
       /manage/       → household editor (no service worker; old Manage SW caches are cleared on load)

  f) Optional custom domain: Amplify → Domain management → add your domain.


2) Supabase sync via Amplify (optional)
---------------------------------------
You can avoid committing shared/sync-config.json by defining environment
variables on the Amplify app:

  • SUPABASE_URL      — Project URL (https://xxxxx.supabase.co)
  • SUPABASE_ANON_KEY — Project anon public key

Amplify → App settings → Environment variables → add both.

On the next build, amplify.yml preBuild writes household-console/shared/sync-config.json
into the artifact. TVs and browsers still need the same household sync key
entered in Manage (that key is not in Amplify; it stays per device or you
distribute it privately).


3) Amazon Developer Console (Fire TV / Silk)
--------------------------------------------
This is separate from AWS Amplify. Use it if you submit a hosted web app or
use official testing tools.

  a) Create / sign in to an Amazon developer account:
       https://developer.amazon.com/

  b) For testing on a Fire TV without store submission, use Web App Tester
     (Amazon documentation: "Debug Your Web App" / Web App Tester). Point it
     at your Amplify HTTPS URL + /tv/

  c) For a store-listed hosted web app, follow Amazon's "Getting Started with
     Web Apps (Fire TV)" — you will provide the production HTTPS URL, icons,
     and metadata in the developer portal.

Silk on Fire TV: you can also simply bookmark the same /tv/ URL; no developer
console is required for household use.


4) Quick checklist
------------------
  [ ] Amplify app created, build green, HTTPS URL works for /tv/ and /manage/
  [ ] (Optional) SUPABASE_URL + SUPABASE_ANON_KEY set in Amplify; rebuild
  [ ] sync-config.json present in deployed site (from build or manual upload)
  [ ] Same household sync key saved on TV (if using cloud) — set in Manage
  [ ] Fire TV: Silk full-screen bookmark or Web App Tester URL = …/tv/


6) Silk on Fire TV: "won't connect" / page errors
---------------------------------------------------
Narrow it down:

  • Same URL on a phone or laptop (same Wi‑Fi): if it fails everywhere, the
    site or DNS is wrong; if it works only off the TV, the issue is the Fire TV
    or Silk.

  • Use a full HTTPS URL in the address bar, including the path, for example:
      https://YOUR-ID.amplifyapp.com/tv/
    Try both with and without a trailing slash. Avoid http:// for production.

  • In AWS Amplify Console: confirm the last deploy succeeded (green). Open
    the exact domain Amplify shows (main branch URL).

  • Fire TV: Settings → My Fire TV → About → check system time is correct
    (wrong date/time can break HTTPS / certificate checks).

  • Wi‑Fi: guest networks sometimes isolate devices from the internet; try
    another network or turn off guest isolation.

  • Silk: Settings → Privacy → clear browsing data (cache/cookies) for the
    site, then try again.

  • Custom domain: wait for DNS to propagate; verify SSL in Amplify Domain
    management shows "Available".

If the page is blank but "connected", open /tv/ on a desktop and use browser
devtools — rare script errors; this app loads scripts from the same host only.
